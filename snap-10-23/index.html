Types of branches in Git Flow

  • Master Is never developed on, has the release branch merged into it after proper testing.
  • Development/Integration is never developed on, has the active ticket branches merged into it when completed, and merges into release branch after a little bit of debugging
  • Active Ticket BranchThe only branches that are actively developed on, are checkedout when a new ticket is generated and are merged into integration branch when completed.
  • Release Test BranchIntegration branch is merged into this towards the end of a sprint for final thorough testing and bugfixing before merging to the master branch
  • HotfixShould never be used except in DIRE situations. checked out solely to fix a single critical error then is merged into master.
